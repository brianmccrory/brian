* June 2025

This change originally started as an exploratory project, to see what it would take to migrate from Markdown format to Emacs Org-mode.

My main goal was to simplify my writing environment by moving from editing Markdown files in Eclipse to editing Org-mode files in Emacs.

This also allowed me to improve my manual Substack import process. Instead of copy and pasting rendered HTML pages Hugo and Markdown files, I could generate my rendered output through the Emacs Org-mode export. This would mean that Hugo was no longer a piece of my writing/publishing process.

What changed:

- Migrated writing environment from Hugo to Org-mode files
- Replaced legacy Java program with Python scripts
- see README-ORG-MIGRATION.md

M-x eval-defun

#+BEGIN_SRC lisp
(defun copy-id-to-export-file-name ()
  "Copy :ID: property to :EXPORT_FILE_NAME: property for each top-level Org subtree.
Appends '.md' to the ID value for EXPORT_FILE_NAME."
  (interactive)
  (org-map-entries
   (lambda ()
     (let ((id (org-entry-get nil "ID")))
       (when id
         ;; Set EXPORT_FILE_NAME to ID + ".md"
         (org-entry-put nil "EXPORT_FILE_NAME" (concat id ".md"))
         (message "Set EXPORT_FILE_NAME=%s.md" id))))
   "LEVEL=1" ;; adjust if you want other heading levels
   'file))
#+END_SRC

2. export: C-c C-e H A

3. remove the new properties:

#+BEGIN_SRC lisp
(defun remove-properties-from-entries (property-list)
  "Remove all properties in PROPERTY-LIST from every Org entry in the current buffer.

PROPERTY-LIST should be a list of property names as strings, e.g., (\"ID\" \"EXPORT_FILE_NAME\")."
  (interactive
   (list
    ;; Prompt user to input comma-separated property names, split into list
    (let ((input (read-string "Enter properties to remove (comma separated): ")))
      (mapcar #'string-trim (split-string input ",")))))
  (org-map-entries
   (lambda ()
     (dolist (prop property-list)
       (org-entry-delete nil prop)))
   nil 'file)
  (message "Removed properties: %s" (mapconcat #'identity property-list ", ")))
#+END_SRC
